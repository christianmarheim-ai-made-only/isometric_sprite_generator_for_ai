# Sprite pipeline hardening roadmap

> Produced by a **read-only** multi-agent survey of `isometric_sprite_generator_for_ai`: 9 subsystem
> mappers, each finding adversarially **verified against the source** (the contract is verified at
> runtime, not compiled, so claims had to be exact), then a completeness critic. **83 findings, 0
> refuted, 17 sharpened.** This is a planning overview -- scope for us, a work-list for the AIs that
> will implement it. Every catalog item cites `file:line` evidence. Nothing was changed to produce this.

## The one reframe that re-orders the work

The output is verified at **runtime**, so a seam only matters if the engine actually **consumes** it.
The dominant finding is a **declare-vs-consume asymmetry**, and its apex is blunt:

> **The R8 hitmask -- the pipeline's central deliverable -- is read by no engine code today.** It is
> schema-REQUIRED, provenance-hashed, and gated five ways, for an artifact `sprite.rs` has no field to
> load.

The same disease runs both directions: producers author fields **no baker reads** (`forward_axis`,
named sockets, `region_by_bone`, `spell_orbits`, `markers`, `variant_class`, `physical_metrics`), and
the pipeline **certifies a contract the engine ignores** (`contract_hash`, the `expected_facing` table,
any palette beyond color). The most expensive case: the lockfile's 8-key `{shield,weapon,gear}` palette
+ ADR-0006 "topmost-surface occlusion" semantics are certified by `contract_hash` but **implemented by
no baker and read by no engine** -- `contract_hash` currently certifies fiction.

**Planning consequence: resolve the consume side FIRST (Phase 0).** Hardening or gating a seam the
engine does not read is certifying fiction. Decide what is real, then gate it.

## Three cross-cutting themes

1. **Declare-vs-consume asymmetry** is the master pattern -- on BOTH sides. Producer-inert fields are
   only half of it; the engine ignoring the whole hitmask, the facing table, and the lock seam is the
   other (larger) half.
2. **The lockfile certifies a region model that is not built.** `sprite_contract.lock.json`'s 8-key
   palette + ADR-0006 occlusion are aspiration; production code knows only 4 body regions with no
   z-occlusion resolution. Every gear/weapon character would mis-region silently the moment a consumer
   appears.
3. **Runtime-visible-but-gate-invisible errors funnel to one missing safety net.** `forward_axis`
   rotation, lying-down quadrupeds, collapsed regions, mis-textured skins, and unit/scale blowups each
   produce a *structurally valid* manifest and a *wrong picture*. The contact sheet that would catch
   them is opt-in and best-effort. Because the contract is runtime-verified, these are only ever caught
   by eyeballing the running engine.

## Phasing (sequence, not a schedule)

**Phase 0 -- Decide the consume side (unblocks the rest; mostly ADRs + the engine team).**
The hitmask status (deferred vs wire a consumer), the lockfile palette / ADR-0006 occlusion vs 4-region
reality, and the version handshake (engine reads no `contract_hash` / `manifest_version`; vendored
schema is stale vs the authoritative engine copy; `terrain` `variant_class` is a 3-way drift). Until
these are decided, gating is certifying fiction. *Findings: C (runtime-seams), B (schema-seams), and the
"Beyond the 83" hitmask/palette/occlusion items.*

**Phase 1 -- Close the runtime-visible failure class (the gate-invisible bugs).** Make a **required,
hashed, human-reviewable contact sheet** part of the batch trust signal -- the single net for the whole
class. Land the spawned `forward_axis` live correction + a per-package facing oracle (embed
`expected_facing` in the manifest, engine cross-checks). Extend the upright/orientation detector beyond
bipeds. Promote region-collapse + all-collapse to a blocking gate. Add a world_metrics scale/units
sanity band. Honor `geometry.up` on the glb path. *Findings: A, C, G, plus the scale/eye-height misses.*

**Phase 2 -- Formalize the producer<->pipeline seams (stop advertising inert capability).** Schemas +
gates for the unvalidated sidecars (`package_manifest`, `sockets`, `spell_orbits`, `materials`,
`physical_metrics`) and **rig profiles**. Decide each inert field -- *implement or delete from schema*:
`forward_axis`, named sockets (3D->2D per-frame projection: the weapons/VFX seam), `region_source`
variants, `variant_class` propagation, `directional`, `metric_proxy_objects`, `region_by_bone`,
`markers`. Apply declared regions on the **pre-rigged** path too (today only auto-rig honors them). Give
the textured-art path a manifest/provenance field (UV + source-texture identity). *Findings: A, D, E, F.*

**Phase 3 -- Stabilize for scale / batches.** Make `bake_batch` a committed end-to-end gate on a real
delivery; emit a durable batch-result + cross-batch inventory artifact; make provenance deterministic
(paths, atomicity, name the *delivered* mesh for auto-rigged packages); abort on oversize atlas. Lock
the Blender version + Python deps so a *new* delivery is byte-deterministic across machines. Add the
vendored-schema drift gate + put the engine schema in the dist; gate the `bevy_reference` loader against
the engine. Single source of truth for the clip vocabulary + a governance gate; an once-terminal golden.
*Findings: G, I, F.*

**Phase 4 -- Effects / color / M2A (when first needed).** Emit the per-creature **core anchor** to the
manifest (orbit + hide-when-behind); represent the synchronous orbit clock + multi-instance phase;
specify the `color_variants` tint table + a recolorable-authoring check; a fat-body radius gate; the
palette-index recolor mask; the `effect` variant_class anchor/depth binding. M2A weapons/gear: regions
5-7, attach sockets, and the ADR-0006 occlusion implementation. *Findings: E, plus M2A in H.*

- **Depends on:** spell-orbits finding (same projection machinery)

#### `anchor-socket-drift-across-states-undetected` -- Anchor + socket consistency ACROSS states is not gated -- a per-state re-measure could drift the foot anchor between idle and walk with no detector
*scope M | artifact: gate/test*

- **Now:** blender_render_anim.py measures one shift/anchor for the whole glb (base_mat at line 67, anchor_frac=probe((0,0,0)) at line 221) from the REST pose, and every frame across every state reuses the same ax,ay (blender_bake.py:208 sets ax,ay once; :216 reuses them per frame). The loop-seam/anchor-drift gate (task 35) guards the frame-0==frame-N seam WITHIN a clip, not the foot anchor's stability ACROSS states, and Gate-1 does not compare anchors between states. world_metrics is measured ONCE from the rest pose, so a state that translates the root (a lunge, a death flop) is not reflected.
- **Gap:** Because all states share one anchor, a state whose animation moves the body's ground contact (death flop, lunge) will have its foot anchor in the wrong place -- the anchor is pinned to the rest pose. There's no detector that the baked silhouette's actual foot-contact matches the declared anchor per state. This is the cross-state analogue of the within-clip loop-seam gate.
- **Why now:** Multi-state packages with translating clips (the cow has death+hit 'once' states) are the norm now. A mis-anchored death state makes the corpse hover or sink at runtime. Formalizing a per-state anchor/footprint sanity check closes the last anchor silent-failure class.
- **Seam:** per-state foot anchor vs the baked silhouette's actual ground-contact row -> engine draw origin
- **Runtime impact:** Engine places the sprite by its anchor; a stale anchor on a translating state makes the entity visually detach from its ground position at runtime (hover/sink), undetected at load.
- **Depends on:** named-sockets-inert
- **Evidence:** pipeline/tools/blender_render_anim.py:221; pipeline/tools/blender_bake.py:208; pipeline/tools/blender_bake.py:216

## Critic's top priorities (verbatim)

- Formalize that the engine consumes ZERO hitmask today and decide the hitmask's real status (deferred vs. wire a consumer) -- it is schema-REQUIRED, provenance-hashed, and gated five ways for an artifact sprite.rs has no field to read; this silently invalidates the entire region/palette/ADR-0006 investment.

- Give the textured-art path a manifest/provenance seam (source texture + UV identity hashed) -- a full texturing subsystem and dist example exist with zero contract field, so 'is this the right skin, reproducibly' is unanswerable from a shipped package.

- Add an absolute scale/units sanity band to world_metrics and embed (or per-package load) the expected_facing table -- both are runtime-consumed-or-consumable proofs (scale, facing) that currently have no upstream guard, so a cm-authored or rotated delivery is only caught by eye in-engine.
